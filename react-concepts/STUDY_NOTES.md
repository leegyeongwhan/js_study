# 리액트 핵심 개념 정리 (React Concepts)

사용자님이 질문하신 **세 가지 핵심 질문**에 대한 상세한 답변과, 우리가 만든 **실습 코드**에 대한 설명서입니다.

---

## 1. 단방향 데이터 바인딩 (Unidirectional Data Binding)이란?

> **"데이터는 오직 위에서 아래로만 흐른다" (Top-Down)**

리액트의 가장 중요한 **원칙**입니다.

-   **의미**: 부모 컴포넌트(`Parent`)가 자식 컴포넌트(`Child`)에게 데이터를 줄 수는 있지만(`props`), 자식이 부모의 데이터를 직접 수정하거나 거꾸로 올려보낼 수는 없습니다.
-   **이유**: 데이터가 사방팔방으로 흐르면, 나중에 버그가 생겼을 때 "도대체 누가 이 데이터를 바꾼 거야?" 하고 범인을 찾기가 매우 어렵기 때문입니다.
-   **실습 코드 참고**: 모든 예제에서 화살표(<, >)가 부모 안에 자식이 있는 형태로만 되어 있는 것이 그 증거입니다.

---

## 2. Props Drilling (프롭스 드릴링)이란?

> **"데이터를 전달하기 위해, 상관없는 중간 컴포넌트들이 파이프 역할만 하는 현상"**

-   **상황**: `할아버지`가 `손자`에게 용돈을 주려는데, 멀리 살아서 직접 못 줍니다. 그래서 `아버지` → `나` → `손자` 순서로 전달합니다.
-   **문제점**:
    1.  `아버지`와 `나`는 용돈이 필요 없는데도 받아서 전달하는 코드를 짜야 합니다 (귀찮음).
    2.  중간 사람이 많아질수록 코드가 지저분해지고 관리가 힘듭니다.
-   **실습 코드**: `src/components/PropsDrilling.jsx`
-   **해결 방법**:
    1.  **Context API**: 방송국처럼 전역으로 뿌려주기.
    2.  **Store (상태 관리 라이브러리)**: 공용 창고에 넣어두기.

---

## 3. State vs Context Provider vs Store 차이점

가장 헷갈리시는 세 가지 개념을 **'데이터 보관 장소'**로 비교해 드립니다.

### ① Status (State / 지역 상태)
> **"나만의 주머니"**
-   **설명**: 컴포넌트 자기 자신만 관리하는 데이터입니다. (`useState`)
-   **범위**: 나, 그리고 내가 물려준(`props`) 자식까지만 사용 가능.
-   **용도**: 입력창 내용, 버튼 클릭 여부 등 **나한테만 중요한 것**.

### ② Context Provider (전역 상태 / 방송국)
> **"우리 가족 회의"**
-   **설명**: 리액트가 기본으로 제공하는 기능입니다. 특정 범위(`Provider`로 감싼 부분) 안에 있는 모든 컴포넌트에게 데이터를 쏴줍니다.
-   **범위**: `Provider` 태그 하위에 있는 모든 곳.
-   **용도**: 테마 설정(다크모드), 로그인 사용자 정보 등 **자주 안 바뀌지만 여기저기서 필요한 것**.
-   **실습 코드**: `src/components/ContextApiExample.jsx`

### ③ Store (Redux, Zustand / 전역 상태 라이브러리)
> **"마을 공용 창고"**
-   **설명**: 리액트 컴포넌트 바깥에 따로 존재하는 데이터 저장소입니다.
-   **범위**: 앱에 있는 **누구든지** 열쇠(`Hook`)만 쓰면 접근 가능. 부모-자식 관계가 전혀 필요 없습니다.
-   **용도**: 서버에서 받아온 데이터 목록, 장바구니 등 **앱 전체에서 복잡하게 쓰이는 데이터**.
-   **실습 코드**: `src/components/StoreExample.jsx` (Zustand 사용)

---

### 요약표

| 구분 | State (지역) | Context (방송) | Store (창고) |
| :--- | :--- | :--- | :--- |
| **어디서 써요?** | `useState` | `CreateContext` + `Provider` | `create` (Zustand 등) |
| **누가 써요?** | 나만 (자식한테는 줌) | 감싸진 가족들 모두 | 아무나 다 (Global) |
| **언제 써요?** | 간단한 화면 제어 | 다크모드, 언어 설정 | 복잡한 데이터 공유 |
